# プロジェクトおよびサンドボックス払い出しガイド

本ドキュメントは、Google Cloud プロジェクトの新規作成、サンドボックスの払い出し、およびデプロイの手順について記述したものです。

---

## 1. プロジェクトの申請と作成

### 1.1 通常プロジェクト (App Projects)
1.  `governance/admin/factory/inventory.json` を更新します。
2.  PR を作成し、マージされると自動的にプロジェクトが作成されます。

### 1.2 サンドボックス (Sandboxes)
1.  GitHub Actions の **「GCP: Create Sandbox」** ワークフローを実行します。
2.  `sandbox_id` と `github_repo` を入力して実行すると、自動的に `inventory.json` が更新され、プロジェクトが払い出されます。

---

## 2. セットアップ手順 (重要)

プロジェクト作成が完了すると、**Slack に設定ガイド（YAML ブロック）が届きます。**

### 2.1 デプロイ設定の反映
1.  自分のアプリリポジトリにある以下のファイルを開きます：
    - 通常プロジェクト: `.github/workflows/main.yml`
    - サンドボックス: `.github/workflows/sandbox-deploy.yml`
2.  Slack 通知からコピーした内容を、`env:` セクションにそのまま貼り付けます。

```yaml
env:
  # ===============================================================
  # 🧪 Sandbox 設定エリア (Slack からコピペ)
  # ===============================================================
  APP_NAME:             "..."
  ADMIN_PROJECT_ID:     "..."
  ADMIN_PROJECT_NUMBER: "..."
  ID_SANDBOX:           "..."
  # ===============================================================
```

---

## 3. デプロイの実行

本基盤の雛形ワークフローは、**「インフラ（Terraform）」と「アプリプログラム（gcloud等）」** を分離して効率的にデプロイします。

### 3.1 通常デプロイ (`main.yml`)
- `main` ブランチへのプッシュ、または手動実行。
- **本番（prd）デプロイ**: 自動的に管理側の SA に権限が委譲（Handover）され、安全に実行されます。

### 3.2 サンドボックスデプロイ (`sandbox-deploy.yml`)
- `sandbox/**` ブランチへのプッシュ、または手動実行。
- 自分のサンドボックスプロジェクト内だけで、自由に実験が可能です。

---

## 4. ライフサイクルと自動削除

- **サンドボックスの期限**: 作成時に指定した期限（デフォルト 7日）が過ぎると自動削除されます。
- **削除フロー**: ボットが `inventory.json` から定義を削除し、Terraform が物理削除を行います。
- **ゾンビ防止**: 台帳（Git）と実体（GCP）が常に同期されるため、削除後の不要な課金は発生しません。
